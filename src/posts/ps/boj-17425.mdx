#### 시간제한 : 1초

### 문제
두 자연수 A와 B가 있을 때, A = BC를 만족하는 자연수 C를 A의 약수라고 한다. 예를 들어, 2의 약수는 1, 2가 있고, 24의 약수는 1, 2, 3, 4, 6, 8, 12, 24가 있다. 자연수 A의 약수의 합은 A의 모든 약수를 더한 값이고, f(A)로 표현한다. x보다 작거나 같은 모든 자연수 y의 f(y)값을 더한 값은 g(x)로 표현한다.

자연수 N이 주어졌을 때, g(N)을 구해보자.

### 입력
첫째 줄에 테스트 케이스의 개수 T(1 ≤ T ≤ 100,000)가 주어진다. 둘째 줄부터 테스트 케이스가 한 줄에 하나씩 주어지며 자연수 N(1 ≤ N ≤ 1,000,000)이 주어진다.


### 접근

연관된 문제 http://aaronjeong.dev/ps/boj-17427

1. 10만개의 테스트케이스가 생기는데, 이전 글의 O(n) 방식을 사용할 경우(1-1,000,000) 100만 * 10만의 케이스가 생길 수 있으므로, 1초의 시간제한 안에 풀 수 없다.

2. 시간을 줄이려면 memorization(dp)가 필요하겠다는 생각을 했다. (진행중)
