#### 데이터 모델링의 중요성 및 유의점
- 중복 : 같은 시간 같은 데이터 제공
- 비유연성 : 사소한 업무변화에 데이터 모델이 수시로 변경되면 안된다. 데이터 정의와 사용 프로세스의 분리
- 비일관성 : 데이터 간 상호 연관 관계에 대해 명확히 정의

#### 데이터 모델링
- 개념적, 논리적, 물리적 데이터 모델링

#### 데이터 독립성 요소
- 외부 스키마 : 개개 사용자가 보는 개인적 DB 스키마
- 개념 스키마 : 모든 사용자 관점을 통합한 전체 DB
- 내부 스키마 : 물리적 장치에서 데이터가 실제적으로 저장되는 DB

#### 데이터 독립성
- 논리적 독립성 : 개념 스키마 변경, 외부 스키마에는 영향을 주면 안된다.
- 물리적 독립성 : 내부 스키마의 변경은, 외부 스키마와 개념 스키마에 영향을 주면 안된다.

#### Mapping(사상) : 상호 독립적인 개념을 연결시켜주는 다리
- 논리적 사상 : 외부 스키마 - 개념 스키마
- 물리적 사상 : 개념 스키마 - 내부 스키마

#### 데이터 모델링의 3요소
- 어떤 것(Things)
- 성격(Attributes)
- 관계(Relationships)

#### 모델링의 특징
- 추상화
- 단순화
- 정확화

#### ERD 작업순서
1. 엔터티 그리기
2. 엔터티 배치
3. 엔터티 관계설정
4. 관계명 기술
5. 관계의 참여도 기술
6. 관계 필수 여부

#### 좋은 데이터 모델의 요소
1. 완전성 : 업무에 필요한 모든 데이터가 모델에 정의
2. 중복 배제 : 하나의 DB내 동일한 사실은 한번만
3. 업무 규칙 : 많은 규칙을 사용자가 공유하도록 제공
4. 데이터 재사용 : 데이터가 독립적으로 설계
5. 의사 소통 : 업무 규칙은 엔터티, 서브타입, 속성, 관계 등의 형태로 최대한 자세히 표현
6. 통합성 : 동일한 데이터는 한번만 정의 후, 참조를 활용한다.

#### 엔티티란
- 업무에 필요한 유용한 정보를 저장하고, 관리하기 위한 집합. 보이지 않는 개념을 포함한다.

#### 엔티티의 특징
1. 반드시 해당 업무에서 필요하고 관리하고자 한다.
2. 유일한 식별자에 의해 식별 가능
3. 두개 이상의 인스턴스의 집합
4. 업무 프로세스에 의해 이용되어야 한다.
5. 반드시 속성이 있어야 한다.
6. 다른 엔티티와 최소 1개 이상의 관계 -> 통계성 / 코드성 엔티티는 관계 생략

#### 엔티티의 분류
1. 유무형에 따른 분류
    - 유형 : 물리적 형태 - ex) 사원
    - 개념 : 개념적 정보 - ex) 조직
    - 사건 : 업무 수행시 발생 - ex) 주문
2. 발생 시점에 따른 분류
    - 기본 : 원래 업무에 존재하는 정보, 다른 엔티티의 부모 역할 - ex) 사원
    - 중심 : 기본 엔티티로부터 다른 엔티티로의 관계. 많은 행위 엔티티를 생성 - ex) 주문
    - 행위 : 2개 이상의 부모 엔티티로부터 발생. 자주 바뀌고 양이 증가한다 - ex) 주문 목록

#### 속성(Attribute)
- 한개의 엔티티는 2개 이상의 인스턴스의 집합
- 한개의 엔티티는 2개 이상의 속성
- 한개의 속성은 1개의 속성값을 가짐

#### 속성의 분류
1. 구성 방식의 분류
    - PK
    - FK
    - 파생
2. 일반적 방식 분류
    - 기본 : 업무로부터 추출한 모든 일반적 속성
    - 설계 : 업무를 규칙화하기 위해 새로 만들거나 변형하는 속성 - ex) 일련번호
    - 파생 : 다른 속성에 영향을 받아 발생하는 속성 - ex) 합

#### 도메인
- 속성에 대한 데이터 타입, 크기, 제약사항을 지정한다.

#### 속성의 명명
- 해당 업무에서 사용하는 이름을 부여
- 서술식 속성명은 사용 금지
- 약어 사용 금지
- 구체적으로 명명해 유일성 확보

#### 관계(Relationship)
- 엔티티의 인스턴스 사이의 논리적인 연관성
- 서로에게 연관성이 부여된 상태
- 관계 페어링의 집합

#### 페어링
- 엔터티 안에 인스턴스가 개별적으로 관계를 가지는 것

#### 식별자의 특징
- 유일성 : 주식별자에 의해 모든 인스턴스들이 유일하게 구분
- 최소성 : 주식별자를 구성하는 속성의 수는 유일성을 만족하는 최소의 수
- 불변성 : 주식별자의 값은 자주 변하지 않아야 함
- 존재성 : 주식별자가 지정되면 반드시 값이 들어와야 함
