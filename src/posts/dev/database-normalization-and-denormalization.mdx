### 데이터베이스 정규화와 비정규화의 차이

> 정규화 : 중복을 최소화하여 논리적 흐름에 맞게 변경하는 것

> 비정규화 : Join연산을 줄여 확장성을 확보하기 위해 테이블을 변경하는 것

### 정규화

https://www.aaronjeong.dev/database-normalization

### 비정규화

- 데이터를 의도적으로 중복 배치하여 읽는 연산의 시간을 줄이는 기술

- 의도적으로 정규화 원칙을 위배한다.

- 장점
    1. 빠른 데이터 조회
    2. 살펴볼 테이블이 줄어들기 때문에 데이터 조회 쿼리가 간단해짐

- 단점
    1. 데이터 갱신이나 삽입 비용이 높음
    2. 데이터 갱신 또는 삽입 코드를 작성하기 어려워짐
    3. 데이터 간의 일관성이 깨어질 수 있다. 어느 쪽이 올바른 값인가?
    4. 데이터를 중복하여 저장하므로 더 많은 저장 공간이 필요

- 비정규화 대상
    1. 자주 사용되는 테이블에 액세스하는 프로세스의 수가 가장 많고, 항상 일정한 범위만을 조회하는 경우
    2. 테이블에 대량 데이터가 있고 대량의 범위를 자주 처리하는 경우, 성능 상 이슈가 있을 경우
    3. 테이블에 지나치게 조인을 많이 사용하게 되어 데이터를 조회하는 것이 기술적으로 어려울 경우
    4. 대부분의 대규모 IT 업체의 경우처럼, 규모 확장성(scalability)을 요구하는 시스템 경우 거의 항상 정규화된 데이터베이스와 비정규화된 데이터베이스를 섞어 사용한다.

- 주의점
    1. 반정규화를 과도하게 적용하다 보면 데이터의 무결성이 깨질 수 있다.
    2. 입력, 수정, 삭제의 질의문에 대한 응답 시간이 늦어질 수 있다.
