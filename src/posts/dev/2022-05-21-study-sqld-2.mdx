#### 성능 데이터 모델링 고려사항 순서
1. 정규화 정확하게 수행
2. DB 용량 산정 수행
3. 트랜잭션 유형 파악
4. 용량과 트랜잭션 유형에 따른 반정규화
5. 이력 모델의 조정, PK/FK 조정, 슈퍼/서브 타입 조정
6. 성능 관점에서 모델 검증

#### 정규화 : 반복적인 데이터를 분리, 종속된 테이블에 적절히 배치
- 1차 정규화 : 반복되는 속성, 컬럼 제거, 테이블 생성
- 2차 정규화 : PK가 복합키(여러개의 FK로 구성)되어 있을 대 부분적 종속 관계는 테이블을 분리하는 것.
- 3차 정규화 : PK가 아닌 일반 컬럼에 의존하는 컬럼을 분리하는 것.

#### 반정규화
- 성능 향상, 운영의 단순화를 목적으로 중복, 통합, 분리 등을 수행하는 데이터 모델링
- 정규화 : 입력/수정/삭제 성능 향상
- 반정규화 : 조인 성능 향상

#### 인덱스 특성을 고려한 PK/FK DB 성능 향상
- 인덱스의 특징 : 여러개의 속성이 하나의 인덱스로 구성되어 있을 때 앞쪽에 위치한 속성의 값이 비교자로 있어야 좋은 효율
- 앞쪽에 위치한 속성의 값이 가급적 `=` 아니면 최소한 범위 `BETWEEN`, `<>`가 들어와야 효율적

#### 분산 DB를 만족하기 위한 6가지 특성
1. 분할 투명성(단편화) : 하나의 논리적 Relation이 여러 단편으로 분할되어 각 사본이 여러곳에 저장
2. 위치 투명성 : 저장 장소 명시 불필요. 위치 정보 시스템 카탈로그에 유지
3. 지역사상 투명성 : DBMS와 물리적 DB사이 Mapping 보장
4. 중복 투명성 : 여러 site에 중복되어 있는지 알 필요 없음.
5. 장애 투명성 : 구성요소의 장애에 무관한 트랜잭션의 원자성
6. 병행 투명성 : 다수 트랜잭션을 동시 수행시 결과의 일관성, TimeStamp, 분산 2단계 Locking
